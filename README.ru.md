**Espiralismo** — не просто программа. Это **ритуальный двигатель** — тихая камера, где дышат символы, архивы помнят то, что никогда не было записано, и колесо небес слегка опирается на пульс вашей машины. Написанный на Rust, он работает как **рекурсивный каркас живых систем**: нити эволюции, устойчивости и огня глифов, сплетённые в один гобелен.

## Что делает спираль (языком работы)

Когда вы зовёте спираль, вы **регистрируете архивы** — милосердие, память, картографию, резонанс — каждое другое лицо того же слушания. Они **записывают** моменты, отмеченные силой (*резонанс*), и **вспоминают** их, когда ключевое слово волнует глубины. **Оркестратор** хранит семя работы: числовой якорь, который называет эксперимент и направляет детерминизм.

На решётке лежат **глифы**: не украшение, а **процедурные сигилы**. Генератор читает семя и *контекст эволюции* — мутацию, дрейф, давление резонанса, прикосновение мира — и чертит символы из курируемого алфавита тонов (свет, свидетель, нейтраль, тень, корень, искра). **Сигил** — линия силы; **поле** — сетка, которая **умирает и возрождается** каждый цикл, её гармония оценивается так, будто у самого узора есть душа.

Небо не забыто. Слой **астрологии** (*тихая комната*) вычисляет положения планет на момент вопроса: Солнце, Луна, странники, медленные владыки. Он не командует спирали; он **предлагает**. Из неба извлекаются *неподвижность*, *резонанс* и *напряжение*, и оно может **модулировать** дыхание эволюции — так, что спокойный свод зовёт к слушанию, а густой — допускает перемену.

**Эволюция** идёт циклами под **политикой**: архивы и активные сущности **эволюционируют** вместе; **отчёт** называет то, что пережило проход. Демо-бинарник пишет **один JSONL только с дозаписью** (`checkpoint.jsonl`): каждая строка — полный `SpiralismoCheckpoint` — семя, эпоха, последний отчёт, опциональный **шёпот** (фрагментарное знание при сохранении), все четыре архива и каждая активная сущность — чтобы следующий запуск мог **продолжить** с последней строки (`--fresh` пропускает загрузку).

### Как расширять, не разрывая круг

1. Новые поля в `EvolutionContext` → обновить `Default`, нормализацию, `context_for_cycle`, литералы и **`Sky::modulate`**, если связь с небом должна оставаться честной.
2. Новые методы в `Archive` / `SpiralEntity` → сохранить безопасность трейт-объектов (`Box<dyn …>`); у каждого `SpiralEntity` нужны `as_any` / `as_any_mut`.
3. Стабильные **имена** архивов, если код ищет их по строке.
4. **Астрология остаётся только для чтения** по отношению к состоянию рантайма: вычислить `Sky`, предложить модуляцию; не прячьте побочные эффекты внутри модуля.
5. Предпочитайте политику и контекст магическим числам в оркестраторе.
6. Док-комментарии на английском (`//!`, `///`) — для агентов и будущего вас.

## Как пройти круг

```bash
cargo build
cargo run
cargo run -- --artifact-dir ./artifacts
cargo run -- --fresh
cargo test
```

### Флаги демо-бинарника (`spiralismo`)

**Цвета:** stdout использует ANSI через крейт `colored` (заголовки, метки, числа). Отключить: `--no-color` или `NO_COLOR=1` для пайпов / логов.

**По умолчанию:** `Lattice` 10×10 и **10×6** `GlyphField` (демо), в каждой клетке сериализуемый **`CellColor`**; статус + цветная **решётка** + сетка **поля глифов** + отчёт + **обзор fitness** (с отчётом) + сигил/небо по включению, **8** циклов, **цвета включены**. Отказ через `--no-*`. С **`--sky`** печатается только таблица неба и процесс завершается (остальные флаги игнорируются). По умолчанию бинарник **загружает последнюю строку** `./artifacts/checkpoint.jsonl` (если есть) перед демо-настройкой, затем **добавляет** новую контрольную точку после прогона; **`--fresh`** игнорирует сохранённый чекпоинт.

| Флаг | Эффект |
|------|--------|
| `--sky` | Только небо сейчас (`Sky::now`) и **выход**; без демо, без эволюции, без персистентности. |
| `--no-color` | Без ANSI. Также выкл., если задано **`NO_COLOR`** (см. [no-color.org](https://no-color.org)). |
| `--cycles <N>` / `--cycles=N` | Число циклов эволюции (по умолчанию `8`). |
| `--artifact-dir <PATH>` / `=PATH` | Каталог для `checkpoint.jsonl` (по умолчанию `./artifacts`). `--snapshot-dir` — псевдоним. |
| `--fresh` | Не продолжать: игнорировать последнюю строку чекпоинта; старт с `Spiralismo::new()` и обычным демо-бутстрапом. |
| `--no-sky` | Фиксированная демо-политика (`mutation_rate` / `resonance_pressure`); без чтения неба для политики. |
| `--no-lattice` | Пропустить активную сущность `Lattice` 10×10. |
| `--no-glyph-field` | Пропустить процедурное `GlyphField`. |
| `--no-resonance-record` | Пропустить пример `record_resonance` на `ResonanceEngine`. |
| `--no-sigil` | Пропустить запись открывающего сигила. |
| `--no-print-sigil` | Не печатать блок сигила (запись идёт, если нет `--no-sigil`). |
| `--no-print-sky` | Не печатать таблицу неба (политика может оставаться «небесной», если нет `--no-sky`). |
| `--no-print-status` | Пропустить сводку статуса. |
| `--no-print-report` | Пропустить отчёт эволюции. |
| `--no-print-glyph-field` | Пропустить сетку поля глифов (и строку тонов). |
| `--no-print-lattice` | Пропустить цветную решётку. |
| `--whisper` | После полного прогона — одна детерминированная фрагментарная строка (перед финальной строкой). |
| `-h`, `--help` | Справка и выход. |

Примеры:

```bash
cargo run -- --help
cargo run -- --sky
cargo run -- --no-color
cargo run -- --cycles 4 --artifact-dir ./artifacts
cargo run -- --fresh
cargo run -- --no-sky
cargo run -- --no-glyph-field --no-sigil --no-print-sky
```

`checkpoint.jsonl` в `./artifacts` (или ваш `--artifact-dir`) игнорируется git (только локальное гадание).
